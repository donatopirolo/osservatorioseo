# Tracker v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the tracker from a data-centric 5-section dashboard (v2) to a 10-section strategic intelligence report organized by SEO questions, with IT vs World side-by-side, hero section, KPI cards, and interpretation text.

**Architecture:** Backend changes are minimal — add device_type and OS data to models/client/collector (+4 API calls). Frontend is a complete rewrite of the Jinja template and JS chart engine: 10 question-based sections, side-by-side IT/World charts, dynamic hero and KPIs, editorial interpretation under each chart.

**Tech Stack:** Python 3.12, Pydantic v2, httpx, Jinja2, Tailwind CSS, vanilla JS (SVG charts).

**Spec:** `docs/superpowers/specs/2026-04-13-tracker-v3-design.md`

```python
    async def device_type_timeseries(
        self,
        *,
        location: str | None = None,
        date_range: str = "12w",
    ) -> list[dict]:
        """Fetch mobile/desktop/other % timeseries.

        Returns list of {date, mobile_pct, desktop_pct}.
        """
        params: dict[str, Any] = {"dateRange": date_range}
        if location is not None:
            params["location"] = location
        data = await self._get("/http/timeseries_groups/device_type", params)
        serie = data["result"].get("serie_0", {})
        timestamps = serie.get("timestamps", [])
        mobile = serie.get("mobile", [])
        desktop = serie.get("desktop", [])
        return [
            {"date": ts, "mobile_pct": float(m), "desktop_pct": float(d)}
            for ts, m, d in zip(timestamps, mobile, desktop, strict=False)
        ]

    async def os_summary(
        self,
        *,
        location: str | None = None,
        date_range: str = "28d",
    ) -> list[dict]:
        """Fetch OS distribution. Returns [{os, pct}] sorted desc, other last."""
        params: dict[str, Any] = {"dateRange": date_range}
        if location is not None:
            params["location"] = location
        data = await self._get("/http/summary/os", params)
        summary = data["result"].get("summary_0", {})
        other_pct = summary.pop("other", None)
        rows = sorted(
            [{"os": k, "pct": float(v)} for k, v in summary.items()],
            key=lambda r: r["pct"],
            reverse=True,
        )
        if other_pct is not None:
            rows.append({"os": "other", "pct": float(other_pct)})
        return rows
```

```python
    # ------------------------------------------------------------------
    # Section 9: device type + OS
    # ------------------------------------------------------------------

    async def _fetch_device_type_both(
        self,
    ) -> tuple[DeviceTypeTimeseries, DeviceTypeTimeseries]:
        it = await self._safe(
            self._fetch_device_type,
            "device_type(IT)",
            location="IT",
            default=DeviceTypeTimeseries(),
        )
        glb = await self._safe(
            self._fetch_device_type,
            "device_type(global)",
            location=None,
            default=DeviceTypeTimeseries(),
        )
        return it, glb

    async def _fetch_device_type(self, location: str | None) -> DeviceTypeTimeseries:
        raw = await self._radar.device_type_timeseries(location=location)
        points = [
            DeviceTypePoint(
                date=_parse_dt(p["date"]),
                mobile_pct=p["mobile_pct"],
                desktop_pct=p["desktop_pct"],
            )
            for p in raw
        ]
        return DeviceTypeTimeseries(points=points)

    async def _fetch_os_both(self) -> tuple[list[OSEntry], list[OSEntry]]:
        it = await self._safe(self._fetch_os, "os_summary(IT)", location="IT", default=[])
        glb = await self._safe(self._fetch_os, "os_summary(global)", location=None, default=[])
        return it, glb

    async def _fetch_os(self, location: str | None) -> list[OSEntry]:
        raw = await self._radar.os_summary(location=location)
        return [OSEntry(os=r["os"], pct=r["pct"]) for r in raw]
```

### Task 5: Rewrite JS chart engine for v3

**Files:**
- Modify: `site/tracker/tracker-charts.js`

Complete rewrite. The JS must render:

1. **Hero** (`#hero`): Find the AI platform with the best rank in `ai_platforms_it`. Show: "{domain} è tra i {rank} siti più popolari in Italia". If no AI in top 100, show the best bucket.

2. **4 KPI cards** (`#kpi-cards`): Each card is a `<div>` with value + label. Values:
   - Best AI platform rank in IT (dynamic)
   - User Action % from last crawl_purpose_it point
   - Bot % from last bot_human_it point
   - Top agent name from last ai_bots_ua_it point

3. **Section 1 chart** (`#chart-s1`): Line chart of AI platforms that have been in top 100 (from top10_it, filtered to domains that appear in ai_platforms_it). Plus ranking table (`#table-s1`) of all 17 platforms with IT rank/bucket, Global rank/bucket, gap indicator.

4. **Section 2 charts** (`#chart-s2-it`, `#chart-s2-global`): Side-by-side stacked area charts for crawl purpose. Colors: User Action=#00f63e, Training=#e24c4c, Mixed=#f5a623, Search=#4ca6e2, Undeclared=#919191. With legend explaining each purpose.

5. **Section 3 charts** (`#chart-s3-it`, `#chart-s3-global`): Side-by-side multi-line charts for AI bot user agents. Clickable legend.

6. **Section 4 chart** (`#chart-s4`): Horizontal bars IT (green) vs World (grey) for 6 key metrics.

7. **Section 5 charts** (`#chart-s5-it`, `#chart-s5-global`): Side-by-side line charts showing only training bots (GPTBot, ClaudeBot, Meta-ExternalAgent, Bytespider). Filter from ai_bots_ua data.

8. **Section 6 charts** (`#chart-s6-it`, `#chart-s6-global`): Side-by-side stacked area charts showing all bot agent shares over time.

9. **Section 7 chart** (`#chart-s7`): Horizontal bars IT vs World for industry.

10. **Section 8 charts** (`#chart-s8-it`, `#chart-s8-global`): Side-by-side area charts for bot vs human.

11. **Section 9 charts** (`#chart-s9-it`, `#chart-s9-global`): Side-by-side area charts for device type (mobile vs desktop). Plus OS bar chart (`#chart-s9-os`) IT vs World.

12. **Dynamic text** for each section (`#text-s1` through `#text-s9`): Generated from data, following spec templates. Uses template literals with actual values.

Key technical patterns:
- All charts: `<svg viewBox="0 0 W H">` with `class="w-full"`
- Side-by-side: parent `<div class="grid grid-cols-1 lg:grid-cols-2 gap-6">` already in template, JS fills each half
- Colors: `["#00f63e","#f5a623","#e24c4c","#4ca6e2","#e24ce2","#4ce2c4","#e2e24c","#a64cf5","#f54c8a","#8af54c"]`
- SVG creation: `document.createElementNS("http://www.w3.org/2000/svg", tag)`
- Data source: `window.__TRACKER_DATA__`
- Bucket display: rank if present → `#${rank}`, else bucket < 200000 → `tra i primi ${bucket}`, else → `oltre 200.000`
- "Dati non disponibili" fallback in each chart container when data is empty
